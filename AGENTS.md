Working agreements

Always use Git for version control.
Write descriptive commit messages.
Apply Clean Code principles.
Follow naming standards for variables and functions.
Run unit tests before commit.
Write clear code documentation.
Perform code review with team members.
Ensure data security (Security Best Practices).
Validate all user input.
Handle error and exception properly.
Avoid hardcoding sensitive values.
Keep credentials and API Keys confidential.
Apply DRY (Don't Repeat Yourself) principles.
Optimize code performance and database queries.
Use separate environments (Dev, Staging, Prod).
Update dependencies routinely.
Perform code refactoring periodically.
Comply with third-party software licenses.
Backup database before major changes.
Follow the established system architecture.
Log system activities for auditing.
Avoid unused code (Dead Code).
Ensure browser or device compatibility.
Use configuration based on environment variables.
Comply with data protection regulations (such as GDPR).
